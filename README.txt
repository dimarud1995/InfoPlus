Microsoft Visual Studio Community 2019 Version 16.9.2
Visual Studio Code 1.54.3
MySQL Comunity Server 8.0.20 (Workbench 8.0)
.NET Core 3.1, 
EntityFrameworkCore 3.1
Node.js 12.18.3
@vue/cli 4.5.11

Server:
 
1.Поміняти ConnectionString в файлі "Server/InfoPlus/appsettings.json", 
	(Щоб використати іншу БД потрібно додатково встановити NuGet пакети EntityFrameworkCore потрібного провайдера БД,
	і в файлі Startup.cs змінити підключення сервісів)
2.Запустити сервер і перейти за адресою 
		https://localhost:44324/api/koatuu/uploaddb
	після чого дочекатись напису "Saved!" у вікні браузера. Дані імпортовані в бд.
	Якщо по якихось причинах не імпортувались дані то в корневій папці проекту є витяжка данних з бази 
	"dump/koatuudb_koatuu.sql"

Client:
3. Відкрити проект Vue.js ( в папці Client/koatuu-client) та запустити сервер vue.js командою з терміналу "npm run serve".
	(Якщо не встановлені якісь пакети чи node.js, то потрібно встановити. Node.js скачати з офійійного сайту,
	а пакети встановити командою "npm install" з каталогу koatuu-client)
4. Перейти за вказаною в консолі адресою.
	
	Якщо запити не доходять, перевірити чи правильно вказані порти, на яких запущені сервери, 
	якщо порти вказані в запитах до апі бек-енду відрізняються від порту, на якому запущений сервер бек-енду, 
	то в коді фронт-енду в запитах вказати правильний порт, на якому запущений сервер бек-енду.
